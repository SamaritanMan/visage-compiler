Where can I download the OpenJFX Compiler?

The OpenJFX Compiler is in its earliest stages and is not yet fully functional, so there are currently no binaries available for download. You may use the project's Subversion repository to download the project source code and build it.  The current compiler only compiles a subset of the JavaFX Script language, however.

If the OpenJFX Compiler isn't ready yet, why is this project open?

Creating the first compiler for a language involves specifying the language more fully than may be required by an interpreted scripting engine.  We want to open the refinement and specification of the JavaFX Script language, as well as the development of its compiler, to the OpenJFX community members who are interested in helping.

What is the relationship between javac (JDK's Java compiler) and the OpenJFX compiler?

The OpenJFX Compiler extends the javac compiler, providing a new lexer, parser, syntax trees, etc.  This is done to leverage the engineering years of effort that has already been invested in javac.

What are the open JavaFX Script language issues?

They are listed in the OpenJFX Compiler project's issue database, under the "specification" subcomponent.  The list is incomplete, but is growing as we uncover new areas that need specification.

What discussion has there been regarding these language issues?

The OpenJFX Compiler wiki contains notes from our engineering meetings, which we will expand upon.  We welcome developer comments regarding all language and compiler issues, either via the dev@openjfx-compiler.dev.java.net email alias, or in the wiki.

When debugging or profiling the compiler, how do I reference javac sources?

First, you need a copy of the OpenJDK source code, which contains the javac sources.  The "javac" NetBeans project in the OpenJFX compiler source base is then configured to reference that local OpenJDK directory.  Full instructions are found in the javac/README file.
